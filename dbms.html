Database Concepts - Detailed Overview

Data

Data refers to raw, unprocessed facts or figures. It can be numbers, characters, images, or any other outputs from devices or systems. On its own, data has no inherent meaning until processed or interpreted.

Information

Information is processed data that has been organized or structured such that it has meaning and is useful for decision making, analysis, or communication.

Features of Information

  • Accuracy: Information must be correct and free from errors.
  • Relevance: Must be applicable and useful for the purpose it is intended.
  • Timeliness: Should be available when needed.
  • Completeness: Contains all necessary data points.
  • Consistency: Should be the same across different sources and time.

Database and its Purpose

A database is an organized collection of data stored electronically. It allows efficient storage, retrieval, and management of data. Its main purpose is to handle large amounts of data and provide controlled access to multiple users.

Terminologies in Database

  • Table: A collection of related data entries organized in rows and columns.
  • Field: A single attribute or column in a table, representing a data category.
  • Record: A single row in a table containing related data fields, also called a tuple.
  • Tuple: Another name for a record or row in a table.
  • Object: In object-oriented databases, an entity that contains both data and methods.
  • Keys: Attributes used to uniquely identify records (e.g., Primary Key, Foreign Key).

Data Dictionary

A data dictionary is a centralized repository that contains descriptions of data objects or items within the database, including their characteristics, relationships, and constraints. It helps maintain data consistency and understanding.

Database Management System (DBMS)

Introduction

A DBMS is software that manages databases. It provides an interface between users/applications and the database, handling data storage, queries, updates, and administration.

Objectives

  • Efficient data storage, retrieval, and manipulation.
  • Maintain data integrity and security.
  • Support concurrent multi-user access.

Advantages

  • Improved data sharing across applications.
  • Enhanced data security and privacy.
  • Minimized data redundancy.
  • Easier data access through query languages.

Disadvantages

  • High initial setup and maintenance costs.
  • Complexity requiring trained personnel.
  • Potential performance overhead.
  • Possibility of data breaches if not secured properly.

Types of Database Models

  • Hierarchical Model: Organizes data in a tree structure with parent-child relationships.
  • Network Model: More flexible than hierarchical, allows many-to-many relationships represented as graph structures.
  • Relational Model: Represents data in tables with rows and columns; most widely used model.
  • Entity-Relational Model: Extension of relational model incorporating entities and relationships, used for designing schemas.

Integrity Constraints and Types

  • Domain Constraint: Specifies allowable values for an attribute (e.g., data type and range).
  • Entity Integrity: Ensures each row has a unique primary key (no null or duplicate keys).
  • Referential Integrity: Ensures foreign keys correctly reference existing entries in related tables.

Keys

Keys uniquely identify records or link tables. Common types are:

  • Primary Key: Uniquely identifies each record in a table.
  • Foreign Key: Attribute in one table that references a primary key in another.
  • Candidate Key: Any attribute(s) capable of being a primary key.
  • Composite Key: A key made up of multiple attributes.

Normalization

Introduction

Normalization is a systematic process of organizing data in a database to reduce redundancy and dependency.

Normal Forms

  • First Normal Form (1NF): Eliminate repeating groups; each field contains atomic values only.
  • Second Normal Form (2NF): Remove partial dependencies; non-key attributes depend fully on the primary key.
  • Third Normal Form (3NF): Remove transitive dependencies; non-key attributes depend only on the primary key.

Advantages

  • Minimizes data redundancy.
  • Improves data integrity and consistency.
  • Eases maintenance and updates.

Disadvantages

  • Normalization can lead to complex joins impacting performance.
  • May complicate queries with multiple table joins.

Centralized and Distributed Database

Introduction

A centralized database is stored and managed at a single location. Distributed databases distribute storage across multiple physical locations.

Advantages

  • Centralized: Easier management and backup.
  • Distributed: Improved reliability, availability, and local performance.

Disadvantages

  • Centralized: Single point of failure, potential bottleneck.
  • Distributed: Complex synchronization and security management.

Comparison

Distributed databases offer scalability and fault tolerance, while centralized databases simplify administration.

Database Security

Introduction

The practice of protecting database systems against unauthorized access, data breaches, or misuse.

Challenges

  • Cyber-attacks and hacking attempts.
  • Managing access control rights.
  • Ensuring data privacy and regulatory compliance.

Security Measures

  • Implementing strong access control and authentication.
  • Encrypting sensitive data.
  • Regular auditing and monitoring.
  • Backup and disaster recovery plans.

Roles of DBA (Database Administrator)

The DBA is responsible for:

  • Installing and configuring the database software.
  • Managing security and permissions.
  • Performing backups and recovery.
  • Monitoring performance and tuning.
  • Ensuring database availability and integrity.

Practical Topics

DDL and DML Languages

Data Definition Language (DDL): Commands to define or modify database structure, e.g., CREATE, ALTER, DROP.

Data Manipulation Language (DML): Commands to manipulate data, e.g., INSERT, UPDATE, DELETE, SELECT.

SQL Data Types Explained with Examples

  • CHAR(n): Fixed-length string.
    Example: 'abc ' for CHAR(5).
  • VARCHAR(n): Variable-length string.
    Example: 'hello' stored as 5 bytes.
  • BINARY(n): Fixed-length binary data.
    Example: 0x4D7953 for binary.
  • VARBINARY(n): Variable-length binary data.
  • TINYBLOB: Small binary object, max 255 bytes.
  • TINYTEXT: Small text, max 255 characters.
  • TEXT: Larger text, max 65,535 characters.
  • LONGTEXT: Very large text, max 4GB.
  • ENUM: Enumeration of values.
    Example: ENUM('small','medium','large').
  • BIT: Bit-field storing 0 or 1.
  • TINYINT: Small integer (-128 to 127).
  • BOOLEAN: True/False (alias for tinyint(1)).
  • INTEGER: Standard integer.
  • FLOAT: Approximate floating point.
  • DOUBLE: Double precision floating point.
  • DECIMAL(p,s): Exact numeric with precision.
    Example: DECIMAL(10,2) for currency.
  • DATE: Date value (YYYY-MM-DD).
  • DATETIME: Date and time (YYYY-MM-DD HH:MM:SS).
