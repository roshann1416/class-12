Unit 1: Database Management System

Data, Information, and Features of Information

Data refers to raw facts and figures without context. It can be numbers, text, images, or other outputs collected for reference or analysis.

Information is processed, organized, or structured data that provides meaning and value. It helps in decision making.

Features of Information:

  • Accuracy: Information must be correct and reliable.
  • Timeliness: Information should be available when needed.
  • Relevance: Information should be applicable to the problem or decision.
  • Completeness: All necessary data should be included.
  • Understandability: Information must be clear and easy to comprehend.

Database and Its Purpose

A Database is an organized collection of data stored electronically. Its primary purpose is to efficiently store, retrieve, and manage large amounts of data.

Databases enable:

  • Efficient data management and retrieval
  • Data sharing among multiple users
  • Data integrity and security
  • Reduced data redundancy and inconsistency

Terminologies in Database

  • Table: A collection of related data entries organized in rows and columns.
  • Field: A single piece of data; a column in a table representing an attribute.
  • Record: A single, complete set of fields; a row in a table.
  • Tuple: Another term for a record or row in a relational database.
  • Object: A data entity that can contain both data and behavior (used in object-oriented databases).
  • Keys: Attributes used to uniquely identify records within a table (e.g., Primary Key).

Data Types

Data types specify the kind of data that can be stored in a field. Common data types include:

  • Integer: Whole numbers without decimals.
  • Float/Double: Numbers with decimal points.
  • Character/String: Textual data.
  • Date/Time: Dates and time values.
  • Boolean: True/False values.

Data Dictionary

A Data Dictionary is a centralized repository that contains definitions, relationships, and attributes of data elements used in a database. It helps maintain consistency and provides metadata information for the database structure.

Database Management System (DBMS)

Introduction

A Database Management System (DBMS) is software that allows users to define, create, maintain, and control access to databases. It acts as an interface between the database and its users or application programs.

Objectives of DBMS

  • To provide efficient and convenient storage, retrieval, and updating of data.
  • To ensure data integrity and security.
  • To minimize data redundancy and inconsistency.
  • To allow multiple users to access data concurrently.
  • To enforce data integrity constraints.

Advantages of DBMS

  • Improved data sharing and data security.
  • Data consistency and reduced redundancy.
  • Efficient data access through query languages like SQL.
  • Backup and recovery facilities.
  • Enforces data integrity constraints.

Disadvantages of DBMS

  • High initial investment and setup cost.
  • Complexity in design and management.
  • Requires specialized personnel (DBA).
  • Performance overhead for simple database applications.

Types of Database Models

  • Hierarchical Model: Data is organized in a tree-like structure with parent-child relationships.
  • Network Model: Uses a graph structure allowing multiple parent and child records, supporting many-to-many relationships.
  • Relational Model: Organizes data into tables (relations) with rows and columns; most widely used model.
  • Entity-Relationship Model: Uses entities and relationships to model data and is often used for database design.

Integrity Constraints and Types

  • Domain Constraint: Ensures data entered in a column falls within a predefined domain or range of values.
  • Entity Integrity Constraint: Ensures primary key values are unique and not null.
  • Referential Integrity Constraint: Ensures foreign keys correspond to valid primary keys in related tables.
  • Key Constraints: Rules that define candidate keys, primary keys, and alternate keys to uniquely identify tuples.

Normalization

Introduction

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity.

Normal Forms

  • First Normal Form (1NF): Ensures atomicity of data; each field contains only indivisible values.
  • Second Normal Form (2NF): Achieves 1NF and removes partial dependency on primary keys.
  • Third Normal Form (3NF): Achieves 2NF and removes transitive dependency.

Advantages of Normalization

  • Eliminates data redundancy and inconsistency.
  • Improves data integrity and organization.
  • Makes database design more flexible and easier to maintain.

Disadvantages of Normalization

  • May lead to complex queries involving multiple tables.
  • Performance overhead due to many table joins.

Centralized and Distributed Database

Introduction

Centralized Database: All data is stored at a single central location and managed from there.

Distributed Database: Data is stored across multiple physical locations but appears as a single database to users.

Advantages

  • Centralized: Easier to manage, better control and security, cost-effective for small systems.
  • Distributed: Improved reliability, faster access for local users, scalability.

Disadvantages

  • Centralized: Single point of failure, slower access for remote users.
  • Distributed: Complex to design and manage, higher costs.

Database Security

Introduction

Database security protects data from unauthorized access, misuse, and breaches.

Challenges

  • Protecting sensitive data from hackers and malicious users.
  • Ensuring data privacy and compliance with laws.
  • Managing user permissions correctly.
  • Preventing data loss due to software or hardware failure.

Security Measures

  • Authentication and authorization mechanisms.
  • Encryption of data at rest and in transit.
  • Regular backups and recovery plans.
  • Auditing and monitoring database access.

DBA and Roles of DBA

Database Administrator (DBA) is responsible for managing and maintaining the database system to ensure availability, security, and performance.

  • Installing and upgrading DBMS software.
  • Defining database schema and structures.
  • Managing database security and user access.
  • Performance monitoring and tuning.
  • Backup, recovery, and data integrity management.

ER Diagrams

Introduction

Entity-Relationship (ER) diagrams visually represent the database structure using entities, attributes, and relationships.

Key Concepts

  • Entity: A real-world object or concept (represented by a rectangle).
  • Attributes: Properties of entities (represented by ovals).
  • Relationships: Associations between entities (represented by diamonds).
  • Cardinality: Specifies the number of instances in one entity related to instances in another (1:1, 1:N, N:M).
  • Degree: Number of entities involved in a relationship.

SQL (DDL and DML)

Data Constraints

  • NOT NULL: Ensures a column cannot have NULL value.
  • UNIQUE: Ensures all values in a column are unique.
  • PRIMARY KEY: Uniquely identifies each row in a table.
  • AUTO INCREMENT: Automatically generates a unique number for new rows.

Operators

  • AND: Combines two conditions; both must be true.
  • OR: Either condition can be true.
  • NOT: Negates a condition.

Where Clause

Used to filter records based on specific conditions.

Order By

Sorts the result-set in ascending or descending order.

SQL JOIN

Used to combine rows from two or more tables based on a related column.

  • INNER JOIN: Returns records with matching values in both tables.
  • LEFT JOIN: Returns all records from left table, and matched records from right table.
  • RIGHT JOIN: Returns all records from right table, and matched records from left table.
  • FULL JOIN: Returns all records when there is a match in either left or right table.

Example of INNER JOIN

SELECT employees.name, departments.department_name
FROM employees
INNER JOIN departments ON employees.department_id = departments.id;
